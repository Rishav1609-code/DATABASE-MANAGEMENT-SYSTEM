ER Diagram from a real-world problem — entities, attributes (incl. multivalued, derived, composite), relationships with cardinality, then draw the diagram. (Direct hit 2023-24 Q7a, similar in 2024-25 Q9b.)

How to answer: Read the problem statement twice. List 3-5 entities first. For each entity, list 3-4 attributes (mark primary key with underline). Connect with lines labelled with cardinality.
Group C — Long Answer (very likely, 15 marks) VERY HIGH

B-Tree insertion problem with a list of integers + a given order. (Came in 2022-23 Q10B and 2023-24 Q7b — the examiner loves rephrasing this.)

How to answer: Start with an empty tree, insert one by one. When a leaf overflows (keys > order-1), split and push the middle key up. Continue until root splits (tree grows taller). Practice 2-3 such insertions: order 3, 4, 5. Use a clean drawing showing the tree at each split step.
Group C — Long Answer (very likely, 15 marks) VERY HIGH

Concurrency control numerical — 2PL with a banking transfer scenario. (2022-23 Q11a and 2023-24 Q11a both did this.)

How to answer: Given T1 and T2 with read/write operations, add lock-X and unlock instructions. 2PL rule: all locks acquired first (growing phase), all locks released later (shrinking phase). Add lock-X(A) before each write, unlock(A) after all reads/writes on A are done. Mention the lock-compatibility matrix.
Group C — Long Answer (very likely, 15 marks) VERY HIGH

SQL + Relational Algebra queries on a given schema — typically 3-4 tables with 6 sub-queries (2-3 marks each). (Direct hit 2024-25 Q11.)

How to answer: For each sub-query, write the relational algebra first (π, σ, ⋈, ÷ symbols), then the SQL. Common operations: select with WHERE, project columns, JOIN 2-3 tables, GROUP BY with aggregate (COUNT, SUM, AVG), subquery for Nth highest, EXISTS / NOT EXISTS. Practice a 3-table schema (Student/Course/Instructor style) with 5-6 sub-questions.
Group C — Long Answer (medium, 15 marks) MEDIUM

ACID properties with example + Dirty read / Phantom read / Non-repeatable read problem. (2023-24 Q2 + 2024-25 Q7 full form.)

How to answer: For each of A/C/I/D — 1 line definition + 1 example. Atomicity: "all or nothing" — bank transfer example. Consistency: valid state before/after. Isolation: concurrent transactions don't interfere. Durability: committed data survives crashes. Then define dirty read: T2 reads uncommitted data from T1 that T1 later rolls back. Use a banking scenario to illustrate.
Group B — Short Answer (almost certain, 5 marks) HIGH

DELETE vs TRUNCATE vs DROP with examples. (Came in 2022-23 Q7a and 2024-25 Q5.)

How to answer: Make a 5-column table: Command | Type (DML/DDL) | Rollback? | Effect | Speed. DELETE = DML, rollback yes, removes rows per WHERE, slow. TRUNCATE = DDL, no rollback, removes all rows, fast. DROP = DDL, no rollback, removes entire table structure + data.
Group C — Long Answer (medium, 15 marks) MEDIUM

Normalization / 3NF decomposition numerical — given R(A,B,C,D,E...) and a set of FDs, decompose into 3NF preserving dependency. (2022-23 Q9a had a 7-mark version with 9 attributes.)

How to answer: Step 1: Find candidate keys using closure of attribute sets. Step 2: For each FD X→Y where X is not a superkey, decompose. Step 3: Remove transitive dependencies. Show each step. End with 3NF relations + verify each is in BCNF. Practice 2-3 such problems — the pattern is the same.

06 — Must-Know Formulae & Rules

The 8 things you must memorise

These are the high-yield "rote-learn" items — examiners love to test these as VSA or as part of a longer answer.

Armstrong's Axioms (FD Inference Rules)
A1. Reflexivity: If Y ⊆ X, then X → Y
A2. Augmentation: If X → Y, then XZ → YZ
A3. Transitivity: If X → Y and Y → Z, then X → Z
B-Tree Node Property (Order m)
Each node: max (m-1) keys, max m children
Min children = ⌈m/2⌉ (except root)
Split rule: when overflow, middle key goes up, two halves stay
2PL (Two-Phase Locking) — The Rule
Growing phase: acquire locks, no release
Shrinking phase: release locks, no acquire
Strict 2PL: all exclusive locks held till commit/rollback
Wait-Die vs Wound-Wait (Deadlock Prevention)
Wait-Die (older waits, younger dies): if TS(Ti) < TS(Tj), Ti waits; else Ti dies
Wound-Wait (older wounds, younger waits): if TS(Ti) < TS(Tj), Ti wounds Tj; else Ti waits
ER Diagram Notation
Rectangle = Entity | Oval = Attribute | Diamond = Relationship
Double oval = Multivalued | Dashed oval = Derived | Underline = Primary key
Double rectangle = Weak entity | Double diamond = Identifying relationship
Normalization Progression
1NF → no multi-valued attributes
2NF → 1NF + no partial dependency
3NF → 2NF + no transitive dependency
BCNF → 3NF + for every FD X→Y, X is a superkey
4NF → BCNF + no multi-valued dependency
5NF → 4NF + no join dependency
Fragmentation Types (DDBMS)
Horizontal: split by rows (SELECT * FROM T WHERE condition)
Vertical: split by columns (each fragment has different attributes + PK)
Hybrid/Mixed: combination of both
Database Security Models
DAC (Discretionary): owner controls access (GRANT, REVOKE)
MAC (Mandatory): system enforces via security labels (Top Secret, Secret...)
RBAC (Role-Based): access based on user's role (Admin, User, Guest)

09 — Exam Hall Tactics

6 things that will save you marks

The paper is designed to be passed. These tips come from how MAKAUT actually grades — they reward structure and diagrams.

01
Always draw ER diagrams
Even for theory questions, a small diagram on the side shows the examiner you understood. For full ER problems — use proper shapes (rectangle, oval, diamond) and label cardinality.
02
Comparison tables for "X vs Y"
DELETE/TRUNCATE/DROP, B-Tree/B+ Tree, DAC/MAC/RBAC, 2PL/Strict 2PL — all comparison questions reward tables. 5-row table beats 3 paragraphs every time.
03
Number your steps in numericals
B-Tree insertion, 3NF decomposition, RA queries — number every step. Even if the final answer is wrong, you get partial marks. Examiners scan for keywords at each step.
04
Skip VSA you don't know
Group A is 1 mark each, you pick 10 of 12. Don't waste 5 minutes on a VSA — move on, come back later. Time is your enemy in 3-hour paper.
05
Pick best 3 of 5 in each group
You don't have to answer the first 3. Skim all 5 questions in Group B and C, then pick the 3 you know best. Many students answer the easiest 3, not the first 3.
06
Banking transaction = 2PL
If a banking transfer (T1, T2) appears, immediately recall the 2PL pattern: add lock-X before each write, unlock after all operations. This is in 2 of 3 years — high chance of repeat.
